# План разработки PWA приложения для изучения английского языка с интеграцией Groq AI

## 1. Исследование возможностей и ограничений

### GitHub Pages для хостинга PWA
- **Преимущества**: Бесплатный хостинг, поддержка HTTPS (обязательно для PWA), простая деплой через Git
- **Ограничения**: Только статический контент (HTML, CSS, JS), нет серверной части
- **Решение**: Все API вызовы к Groq будут выполняться напрямую из браузера клиента

### Groq API (Бесплатные лимиты)
- **Модели**: Llama 3, Mixtral, Gemma и другие открытые модели
- **Лимиты**: 
  - Бесплатный tier: ~30 запросов в минуту (RPM), ~14,400 в день
  - Скорость: Очень высокая благодаря LPU (Language Processing Unit)
- **Использование в PWA**: Прямые вызовы из frontend с хранением API ключа осторожно

### PWA на iPhone (iOS Safari)
- **Поддержка**: iOS 11.3+ поддерживает PWA
- **Особенности**:
  - Manifest файл обязателен
  - Service Worker для offline работы
  - Ограниченный объем Cache Storage (~50-100MB)
  - Push уведомления не поддерживаются на iOS в PWA
  - Добавление на домашний экран через "Share" → "Add to Home Screen"

## 3. Функциональные возможности

### Базовые функции (без AI)
1. **Карточки слов (Flashcards)**
   - Английское слово ↔ Русский перевод
   - Аудио произношение (Web Speech API)
   - Интервальное повторение (Spaced Repetition)

2. **Мини-уроки**
   - Грамматические правила
   - Примеры использования
   - Тесты с множественным выбором

3. **Словарь**
   - Поиск по словам
   - Категории слов (еда, путешествия, бизнес)
   - Избранные слова

4. **Прогресс пользователя**
   - Статистика изученных слов
   - Дневная активность
   - Уровни владения

### Функции с интеграцией Groq AI

1. **AI Репетитор диалогов**
   - Симуляция разговора на заданную тему
   - Исправление ошибок в реальном времени
   - Предложения по улучшению фраз

2. **Генератор упражнений**
   - Создание персонализированных тестов
   - Генерация предложений со словами пользователя
   - Упражнения на заполнение пропусков

3. **Объяснение грамматики**
   - Ответы на вопросы пользователя
   - Примеры использования правил
   - Сравнение с русским языком

4. **Проверка письменных работ**
   - Анализ эссе/предложений
   - Исправление ошибок
   - Рекомендации по стилю

5. **Контекстный перевод**
   - Перевод с объяснением нюансов
   - Варианты использования в разных контекстах

#### Service Worker (sw.js)
- Кэширование статических ресурсов
- Offline режим для базовых функций
- Стратегия: Cache First для статики, Network First для API

### Интеграция с Groq API

#### Безопасность API ключа
⚠️ **Важно**: Поскольку GitHub Pages - статический хостинг, API ключ будет виден в коде. Решения:
1. Использовать прокси-сервер (но это требует бэкенда)
2. Ограничить ключ по домену через Groq dashboard
3. Предупредить пользователей о необходимости своего ключа
4. Использовать переменные окружения при сборке (не подходит для чистого GitHub Pages)

#### Пример вызова API
```javascript
async function callGroqAPI(prompt, apiKey) {
  const response = await fetch('https://api.groq.com/openai/v1/chat/completions', {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
      'Authorization': `Bearer ${apiKey}`
    },
    body: JSON.stringify({
      model: 'llama3-8b-8192',
      messages: [{role: 'user', content: prompt}],
      temperature: 0.7,
      max_tokens: 500
    })
  });
  return await response.json();
}
```

## 6. Рекомендуемые технологии

### Frontend
- **Vanilla JavaScript** или **Vue.js/React** (для простоты можно vanilla)
- **CSS Framework**: Tailwind CSS или Bulma для быстрого прототипирования
- **Хранение данных**: localStorage + IndexedDB для больших объемов

### Библиотеки
- **idb-keyval**: Упрощенная работа с IndexedDB
- **speech-synthesis**: Для аудио произношения
- **chart.js**: Для визуализации прогресса

### Инструменты разработки
- **Vite**: Сборщик для быстрой разработки
- **ESLint + Prettier**: Качество кода
- **GitHub Actions**: Автоматический деплой

## 7. Потенциальные проблемы и решения

| Проблема | Решение |
|----------|---------|
| API ключ в клиентском коде | Пользователь вводит свой ключ, ограничение по домену в Groq |
| Лимиты Groq API | Кэширование ответов, очередь запросов, предупреждения |
| Offline работа AI функций | Сохранение последних ответов, работа только базовых функций offline |
| iOS ограничения PWA | Четкая инструкция по добавлению на домашний экран |
| Объем кэша на iOS | Регулярная очистка старого кэша, приоритизация контента |
